# Phase 2: Pi Runtime Spike

Goal: prove Pi can execute Alfred semantics through the smallest reproducible adapter path.

## Scope

- Implement minimal Pi adapter package code under `packages/pi-adapter`.
- Keep harness-agnostic decisions and data loaders in `packages/core`.
- Load `.ai/manifest.json` and fail fast unless Phase 1 is complete.
- Load the Orchestrator from `.ai/agents/registry.json`.
- Respect the lazy skill registry contract; Phase 2 loads zero skills because no concrete skills exist yet.
- Enforce one permission rule: `orchestrator.read_files` must be `allow` before the eval path runs.
- Run the `schema-validation-local-only` local-first eval path without provider calls.
- Emit a `provider_request_avoided` trace generated by code.
- Keep model assignment user-owned and outside agent specs.

## Runtime Entrypoints

- `packages/core/src/index.js` provides harness-agnostic loading and local-first decision functions.
- `packages/pi-adapter/src/runtime.js` runs the Pi spike path.
- `packages/pi-adapter/src/cli.js` executes the spike from the repository root.
- `scripts/validate-phase-2.mjs` verifies the spike and writes `.ai/observability/generated/phase-2-provider-request-avoided.json`.

## Completion Condition

Phase 2 is complete when `pnpm validate:phase2`, `pnpm check`, and `pnpm test` pass, and the generated trace records `provider_calls: 0`.
