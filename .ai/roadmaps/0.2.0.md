# Roadmap 0.2.0

`0.2.0` moves Alfred from a released architecture kernel to a usable local runtime.

## Phase 8: Runtime Hardening

Turn Pi and opencode spikes into stable runtime APIs with explicit contracts and no harness leakage into core.

## Phase 9: Adapter Generation

Generate harness artifact previews for opencode, Claude, Codex, VSCode, and Kiro. Writes to harness config remain gated by explicit human approval.

## Phase 10: Eval Runner CLI

Expose `packages/evals` through a deterministic CLI that writes local reports and summarizes regressions.

## Phase 11: Release 0.2.0

Package the first usable local runtime release candidate with hardened adapters and executable eval reporting.

## Constraints

- Local-first by default.
- Provider calls allowed for roadmap validation: `0`.
- Core remains harness-agnostic.
- Permission escalation and harness config writes require human approval.
- Baseline updates require human approval.
