Welcome

This console manages API keys and connected services for your Alfred Memory install.

No terminal? No problem. Create a free tenant in 10 seconds.

You'll get an API key immediately. Copy it; we don't store the secret.

Already have an API key?

Paste an API key issued by your Alfred Memory server.

Forgot your API key?

If the email is registered, you'll get a magic link with a new key. The old key will be revoked.

Connected services

Set up ChatGPT, Claude Desktop, or Gemini to use the same memory. Each adapter needs a tenant API key.

1ChatGPT (Plus/Pro)
Import openapi.json as Custom GPT Actions.
Show steps 
1. Run the bridge:
   ALFRED_MEMORY_BASE_URL=<your-server> ALFRED_MEMORY_API_KEY=<key> \
     node packages/chatgpt-adapter/src/bridge.mjs &

2. In ChatGPT: My GPTs → Create → Configure → Actions → Import OpenAPI.
3. Upload: packages/chatgpt-adapter/openapi.json
4. Auth: API Key · Bearer · paste the alk_... key
5. Save and test: ask the GPT "list my last 5 memories"
2Claude Desktop (MCP)
Show steps 
Edit ~/Library/Application Support/Claude/claude_desktop_config.json:
{
  "mcpServers": {
    "alfred-memory": {
      "command": "node",
      "args": ["<abs path>/packages/anthropic-adapter/bin/alfred-mcp.mjs"],
      "env": {
        "ALFRED_MEMORY_API_KEY": "<paste alk_...>",
        "ALFRED_MEMORY_BASE_URL": "<your-server>"
      }
    }
  }
}

Restart Claude Desktop. Tools appear in the conversation panel.

4Coding agents (opencode / Codex / Pi)
Show steps 
1. cd into your project directory.
2. Use the guided installer to create or select a Memory tenant.
3. This writes .alfred/config.json in cwd with your API key.
4. The agent reads it on every session start.
5. Run the Alfred Memory server for that tenant.

What is this?

Alfred Memory is a multi-tenant memory store. The console manages which agents can talk to it and with which credentials.

Memory is never written here — it lives in each tenant's database (Postgres or SQLite). This page only shows the keys and connections.
