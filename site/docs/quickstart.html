5-minute quickstart

From zero to a running agent panel with a passing eval suite, in five minutes. Assumes Node 22+ and one of: opencode, Pi, Codex, Claude, or VSCode.

1. Install

step 1 
curl -fsSL https://raw.githubusercontent.com/GOI17/alfred/main/install.sh | sh

In the TUI, choose coding, keep harness detection on auto unless you know the target, use runtime profiles if this agent will run on more than one machine, and start with preview before applying safe suite steps.

2. Initialize a project

step 2 
cd ~/code/my-project
alfred init --profile=coding --name=my-project
export ALFRED_TENANT=my-project
export ALFRED_API_KEY=alk_...

3. Restart your harness

Configuration is loaded at startup. After alfred init (or any change to .alfred/config.json), restart opencode / Pi / Codex so the panel is registered.

4. Run the eval suite

step 4 
alfred evals run --compare-baseline

Expected output: a list of eval cases, each with a status and a baseline delta. The first run establishes the baseline; later runs compare against it.

5. Talk to the panel

Open a session in your harness and try:

  • "Read AGENTS.md and summarize the kernel." — the orchestrator handles it.
  • "Add a new eval case for the install script." — routes to qa.
  • "Where is the model assignment documented?" — routes to librarian.
  • "Is rm -rf blocked by policy?" — reviewer walks you through .ai/policies/security.md.
Done. You now have a deny-by-default agent panel, a regression suite, and a trace log of every operation. Next: Configuration to pin your model, or Evals & baselines to author your first custom case.
