Codex adapter

Status, install flow, configuration surface, and known limitations of Alfred's Codex adapter.

Status

Adapter status: executable — custom Codex agents and repo skills preview.

Source package: packages/codex-adapter.

Install flow

  1. Run the installer. Alfred detects the active harness.
  2. Review the preview — it lists every file Alfred will write.
  3. Approve. Alfred writes the config and prints a summary.
  4. Restart Codex so the panel is registered.

Configuration surface

The Codex adapter reads .alfred/config.json and:

  • Generates the agent panel under .codex/agent/.
  • Materializes skills under .codex/skill/ with the same triggers.
  • Applies the permission policy unchanged.

Known limitations

  • Configuration is loaded at startup — always restart Codex after alfred init.
  • Protected paths under the harness's own config directory require --force + human approval.
  • Some Codex-specific features (e.g. provider-specific hotkeys) are not yet mapped to Alfred primitives.
Need something missing? Open an issue at github.com/GOI17/alfred/issues with the adapter:codex label.
