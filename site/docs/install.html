Install Alfred

One curl. The installer detects your active harness, generates a preview, and writes harness config only after explicit human approval. No manual git clone.

Heads up. The installer is agent-driven. It only works through an AI agent. The shell script is fetched by the agent, dry-run by default, and waits for your sign-off.

Requirements

  • Node.js 22+ (node -v)
  • pnpm 10+ (optional) — only for monorepo development
  • One of: opencode, Pi, Codex, Claude, VSCode
  • macOS, Linux, or WSL2

Choose a profile

Alfred supports three install profiles. Pick one.

ProfileWhat it doesWhat you need
coding Use Alfred from a coding agent. Local SQLite, no network. Nothing extra.
web Use Alfred Memory from ChatGPT / Claude / Gemini. Postgres URL (Neon, Supabase, Render free tier).
both Use Alfred from coding agents and web UIs. Postgres URL.

Option A — coding profile (no database)

The fastest path. Perfect for trying Alfred.

zsh — install 
# Install
curl -fsSL https://raw.githubusercontent.com/GOI17/alfred/main/install.sh | sh -s -- --profile=coding --name=acme

# Initialize in your project
cd ~/code/my-project
alfred init --profile=coding --name=my-project

# Run the eval suite
alfred evals run --compare-baseline

A SQLite file is created at ~/.alfred/tenants/acme.sqlite. Nothing is uploaded anywhere. The API key is printed once — save it to your password manager.

Option B — web profile (Postgres)

Use Alfred Memory from ChatGPT, Claude, or Gemini.

  1. Create a free Postgres database (Neon / Supabase / Render).
  2. Copy the connection string — it looks like postgres://user:pass@host/db?sslmode=require.
  3. Run the installer. It will prompt for the URL.
zsh — install (web) 
curl -fsSL https://raw.githubusercontent.com/GOI17/alfred/main/install.sh | sh -s -- --profile=web --name=my-mem

Approve, then write

Alfred never writes harness config without your approval. When the installer finishes detection, it prints a preview like:

preview 
# Alfred install preview
Harness detected : opencode
Adapter status   : executable-translation-spike
Files to write   : 7
  + .opencode/agent/orchestrator.md
  + .opencode/agent/developer.md
  + .opencode/agent/qa.md
  + .opencode/skill/architecture-docs.md
  + .opencode/skill/typescript-project.md
  + .opencode/config.json
  + AGENTS.md (merge)

Apply changes? [y/N]

Review the list, type y, and the files land. If you say N, nothing is written and the preview is discarded.

Verify the install

Three commands should pass:

verify 
alfred --version        # prints 0.4.0
alfred doctor         # checks node, pnpm, harness, key
alfred evals list     # shows the regression suite

Uninstall

uninstall 
curl -fsSL https://raw.githubusercontent.com/GOI17/alfred/main/uninstall.sh | sh

The uninstaller removes the installed package tree, never touches your project files, and asks before deleting the SQLite database.
