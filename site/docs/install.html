Install Alfred

One curl. The installer is moving to a suite model: install the coding edition, the Memory edition, or the full Alfred operations system. Harness config is still previewed first and written only after explicit human approval.

Heads up. The installer is agent-driven. It only works through an AI agent. The shell script is fetched by the agent, dry-run by default, and waits for your sign-off.

Requirements

  • Node.js 22+ (node -v)
  • pnpm 10+ (optional) — only for monorepo development
  • One of: opencode, Pi, Codex, Claude, VSCode
  • macOS, Linux, or WSL2

Choose an edition

Alfred installs as a suite of components. Pick the edition that matches the operational surface you need.

EditionWhat it installsWhat you need
coding Core, agents, skills, runtime profiles, opencode/Codex adapters, eval gates. Node 22+ and your coding harness.
memory Alfred Memory, server/client/MCP/OpenAPI, external AI adapters, console. SQLite for local coding agents or Postgres for human/web agents.
full The complete Alfred operations system: coding + Memory components. Node 22+, harness, and Memory storage configuration.

Option A — coding edition

The fastest path. Perfect for coding harness operations without forcing Memory setup.

zsh — install 
# Preview install plan; writes no files
curl -fsSL https://raw.githubusercontent.com/GOI17/alfred/main/install.sh | sh -s -- --edition=coding --name=acme

# Apply only after reviewing the preview
curl -fsSL https://raw.githubusercontent.com/GOI17/alfred/main/install.sh | sh -s -- --edition=coding --name=acme --apply

# Initialize and inspect runtime profiles
alfred-profile init --repo ~/.alfred/profiles
alfred-profile plan --repo ~/.alfred/profiles --profile=work --agent=opencode

# Run the eval suite
alfred evals run --compare-baseline

The coding edition includes packages/profile-manager so the same agent can have separate work, personal, or client profiles across machines without hand-editing harness config.

Option B — Memory edition

Use Alfred Memory from ChatGPT, Claude, Gemini, or coding agents.

  1. Create a free Postgres database (Neon / Supabase / Render).
  2. Copy the connection string — it looks like postgres://user:pass@host/db?sslmode=require.
  3. Run the installer. It will prompt for the URL.
zsh — install (web) 
# Preview Memory edition first
curl -fsSL https://raw.githubusercontent.com/GOI17/alfred/main/install.sh | sh -s -- --edition=memory --name=my-mem

Approve, then write

Alfred never writes harness config without your approval. When the installer finishes detection, it prints a preview like:

preview 
# Alfred install preview
Harness detected : opencode
Adapter status   : executable-translation-spike
Files to write   : 7
  + .opencode/agents/orchestrator.md
  + .opencode/agents/developer.md
  + .opencode/agents/qa.md
  + .opencode/skills/architecture-docs/SKILL.md
  + .opencode/skills/typescript-project/SKILL.md
  + opencode.json
  + AGENTS.md (merge)

Apply changes? [y/N]

Review the list first. The root installer writes no files unless you rerun with --apply; harness config still remains preview-only until a human explicitly copies or applies those generated artifacts.

Verify the install

Three commands should pass:

verify 
alfred --version        # prints 0.4.0
alfred doctor         # checks node, pnpm, harness, key
alfred evals list     # shows the regression suite

Uninstall

uninstall 
curl -fsSL https://raw.githubusercontent.com/GOI17/alfred/main/uninstall.sh | sh

The uninstaller removes the installed package tree, never touches your project files, and asks before deleting the SQLite database.
